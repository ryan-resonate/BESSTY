# `bessty-web`

React + Vite front end for BESSTY. Loads the Rust+WASM solver directly in the browser. Firebase project persistence stub-wired (mock data fallback).

## Status

**v0.6** — full interactive workspace:
- Header with Resonate logo, routed nav.
- Project list with **localStorage persistence**: create new projects, delete, last-updated metadata.
- Project workspace:
  - **Satellite imagery** (Esri World Imagery) base map with one-click toggle to OSM.
  - **Auto-loaded DEM** (AWS Terrain Tiles, free, no key) covering the project area; elevations applied to source/receiver heights and used in the solver.
  - **Draggable** WTGs / BESS / inverters / transformers / receivers — drag-end fires solve and persists the move.
  - Click to place sources or receivers (mode buttons in the side panel).
  - Project wind-speed selector, period selector, per-source model + mode + hub-height edits.
  - **Contour grid rendering** — Run grid computes a coloured raster (Viridis default; 6 palettes available) and overlays it on the map with adjustable opacity.
  - **Glass-chrome layer panel** (top-right) with base-map toggle, contour visibility, opacity slider, palette picker, DEM status.
  - **Legend** (bottom-left) with palette swatches per band and live receiver-count histogram.
  - **Status bar** (bottom-centre) showing project name, wind speed, period, source/receiver counts, current selection.
  - **Results dock** (top-right under layers) with pass/fail bar, exceedance count, worst receiver, and the Run grid button.
  - **Settings modal** (cog icon in side panel) for ground factor, Annex D rules (cap, elevated source, concave correction, receiver minimum), grid spacing.
- **Visual system**: Inter for UI, JetBrains Mono with tabular numerals for measurements, slate neutrals, yellow as accent only, glass chrome with backdrop blur on floating widgets.
- Auto-recompute of point receivers on every state change (debounced ~80 ms); grid is on-demand via Run.

Next deliverables:
- Web Worker wrapping the solver + adaptive contour grid (`docs/architecture.md` §3.3).
- Barrier drawing tool on the map.
- Firestore persistence (mechanical swap from localStorage).
- Annex D.5 concave correction wired through DEM-based hm computation.
- Two-level tabs IA refactor (Sources / Area / Receivers / Import / Results / Layers).

## Run locally

```bash
# 1. Build the solver to WebAssembly (one-off after solver changes)
cd web
npm install
npm run build:wasm     # requires wasm-pack; runs `wasm-pack build` in ../solver

# 2. Start the dev server
npm run dev
# open http://localhost:5173
```

The dev server emits the Cross-Origin-Isolation headers required for SharedArrayBuffer (see `vite.config.ts`). Currently the solver runs on the main thread; SAB matters once the worker layer lands.

## Connecting Firebase

Copy `.env.example` to `.env.local` and fill in the Firebase project values. Until you do, the project list shows mock data and the workspace uses an in-memory demo project.

## Layout

```
src/
  main.tsx              entry
  App.tsx               router
  styles.css            global styles
  components/
    Header.tsx          Resonate logo + nav
    MapView.tsx         Leaflet map, source/receiver markers, click-to-add
    SidePanel.tsx       scenario, add tools, source/receiver lists, Run
  screens/
    ProjectListScreen   project grid + new-project action
    ProjectScreen       map + side panel + auto-solve
  lib/
    firebase.ts         Firebase init (lazy, gated on env vars)
    types.ts            mirrors docs/firestore-schema.md
    projects.ts         listProjects() with mock fallback
    catalog.ts          placeholder WTG / BESS / inverter / transformer spectra
    solver.ts           wraps WASM solver; evaluateProject(p) → per-receiver dB(A)
    demoProject.ts      in-memory sample project for v0.4
  wasm/                 generated by `npm run build:wasm` — do not edit
public/
  ResonateLogo.svg      header brand
```
